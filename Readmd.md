# 翻牌記憶遊戲 (Memory Game)

**作者**: s1141376 Shi  
**版本**: 1.0.0  
**重構版本**: 已按照 OOP 原則進行模組化

## 遊戲規則

1. 利用繼承 Button 的機制來實現翻牌的點擊
2. 每一張牌有背面圖案與正面（撲克牌：A, 2~9, 10, J, Q），共 12 種牌面，每種有兩張，總共 24 張
3. 牌排列為 6 列 × 4 行的網格
4. 一開始時會打散 24 張牌、隨意排列，並顯示背面圖案
5. 玩家點擊牌時會翻出正面，每次最多可以點選兩張牌
6. 若兩張牌的圖案相同，會清除這兩張牌
7. 若圖案不同，會停留 3 秒後翻回背面
8. 遊戲過程中會顯示經過的秒數
9. 當所有牌都被清除後遊戲結束，並詢問是否繼續玩

## 類別說明

### Card 類別 (`card.py`)
- **功能**: 表示遊戲中的單張卡片
- **父類**: `tk.Button`
- **主要方法**:
  - `show_back()`: 顯示卡片背面
  - `show_front()`: 顯示卡片正面
  - `clear_card()`: 清除卡片（配對成功時）
  - `click_card()`: 處理卡片點擊事件

### MemoryGame 類別 (`game.py`)
- **功能**: 管理遊戲的整體邏輯和狀態
- **主要方法**:
  - `create_cards()`: 初始化卡片網格
  - `select_card(card)`: 處理卡片選擇
  - `check_match()`: 檢查兩張卡片是否配對
  - `check_game_finish()`: 檢查遊戲是否完成
  - `update_timer()`: 更新計時器
  - `restart()`: 重新開始遊戲

## 如何執行

```bash
# 方法 1: 直接執行主檔案
python MemoryGameMain.py

# 方法 2: 使用 Python 模組方式
python -m MemoryGameMain
```

## 改進說明

### OOP 模組化優勢
- ✅ **模組分離**: 每個類別獨立成檔案，職責明確
- ✅ **易於維護**: 修改單個類別時不影響其他模組
- ✅ **可重用性**: 其他專案可直接引入 `Card` 或 `MemoryGame` 類別
- ✅ **可擴展性**: 易於新增功能或修改現有類別
- ✅ **代碼可讀性**: 每個檔案只包含相關的邏輯，更容易理解
